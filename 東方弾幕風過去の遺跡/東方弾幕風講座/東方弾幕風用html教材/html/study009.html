第００９回　連弾 (追い越し)

では今回は連弾最後の講座、追い越し型についてだ。

じゃあまずはいつも通りスクリプトを開くところからですね。

いや、今回は先にどんな弾幕になるのか見てみよう。
弾幕風を動かして『連弾 (追い越し)』を選んでみよう。

た、弾がしなるように飛んできます〜！？
これって今までの連弾とは全然性質が違うんじゃないですか？

追い越し弾の最大の特徴はなんといっても見た目のインパクトだな。
だがスクリプトの中身は今までの連弾とほとんど変わらない。
では今度こそスクリプトの中身を見てみようか。
『study009.txt』を開いてみよう。 


〜　前略　〜

  task mainTask {
    loop(60) {yield;}

    loop {
      let speed = 1;
      loop(6) {
        CreateShot01(GetX, GetY, speed, GetAngleToPlayer, RED01, 0);
        speed += 0.5;
        loop(10) {yield;}
      }
      loop(60) {yield;}
    }
  }
}

これって、前回の一斉発射型の連弾のスクリプトとほとんど同じですね。

その通り。
前回のスクリプトと違うところはloop(10) {yield;}が新たに加わっただけだ。
といってもその追加分も最初の２種類の連弾で出てきたな。
つまりこのインパクト大な弾幕は今までの弾幕のちょっとした応用にすぎないワケだ。

それなのに弾幕の印象がこんなにも変わるんですね。

それもまた弾幕作りの面白いところの１つさ。
ちなみにこの弾幕が加速して見える原理だが･･････
この弾幕のミソは最初に撃つ弾ほど遅く、後に出す弾ほど速くなっていることだ。
だから最初は弾が遅く感じるが、後から出た弾がいずれ追い越してくる。
だから弾全体が加速しているように錯覚するワケだな。

だから追い越し弾って読んでるんですね。
そっか〜、弾自体が加速しているワケじゃないのか〜･･････

弾幕の構造自体は極めて原始的であるにも関わらず、視覚的にインパクトのある
弾幕を作れているという素晴らしい弾幕だ。

作り方によってはこんなに面白い弾幕が作れるんですね〜
私もこんな弾幕が作れたらな〜

それは各自の頑張り次第よ。
