閑話休題　インデントについて

また会ったわね。
今回もスクリプト作成に直接的には関係ない話をさせてもらうわ。
･･････といっても、今回は少し真面目に聞いて欲しいわね。
割と大切な話だからね。




さて、今回の話は『インデント』よ。
言い換えると『字下げ』ね。
スクリプトの左側にある空白のことね。
これ何のためにあるのか疑問に思っていた人もいるかもしれないわね。
結論から言うと、意味は無いの。
スペースや改行が無くてもプログラム自体は動くわ。
試しに取っ払ってみましょうか。 


〜　前略　〜
task mainTask {loop(60) {yield;}loop {let angle;angle = GetAngleToPlayer - 20;loop(3) {CreateShot01(GetX, GetY, 2, angle, RED01, 0);angle = angle + 20;}loop(60) {yield;}}}}




これ、なんのスクリプトかわかる？
実はstudy005の『変数を使用した自機狙い3way』のスクリプトそのものよ。
信じられない？
ちなみにこんな書き方でもちゃんと動くわよ。
さっきインデントは意味が無いって言ったけれど、それはプログラム側にとっての話。
スクリプトの中身を見る私達にとっては、見辛いことこの上ないわ。
だからインデントは欠かせないというワケね。



ん？
見辛いのは改行が無いせいで、空白は無くても良いんじゃないかって？
貴方も言うわね。
では別の例で空白だけ取っ払った場合を見てみましょう。
今回の例はstudy010の『5way連弾 (後追い) 』の最後に出た、
変数の寿命で用いた例よ。 


let a = 1;
(処理A1)
loop (2) {
let b = 2;
(処理B1)
loop(3) {
let c = 3;
(処理C)
}
(処理B2)
}
(処理A2)




さて、この書き方でどの { と } が対応しているかわかるかしら？
インデントが無いだけで大分把握しにくくなっていないかしら？
この程度のスクリプトでも見づらくなるのに、もっと複雑なスクリプトだったら･･････
想像するだけで気が滅入るわね。
仮に貴方がこの書き方で慣れたとしても、よ。
他人からアドバイスを貰うために自分のスクリプトを見てもらうとき、
こんな書き方だったら読む気になってもらえないでしょうね。
だから常日頃からインデントを付けるクセを身に付けないとね。



さて、それでインデントの規則なんだけど･･････
一度 { を書いたら、次の行から１段インデントを増やすのね。
そして } で閉じる行で１段インデントを減らすの。
今までのスクリプトを見直すと全てその規則で書かれているのがわかると思うわ。
但しloop(60) {yield;}みたいにループ間の処理が１行で済むような場合は
１行でまとめてしまった方が見やすいかもね。
まあそこは好みの問題で済まされるレベルの話ね。



そして何個の空白を以って１段のインデントとするかについてだけど。
これについては各自の好みね。
毎回バラバラにするのは論外として、半角スペース１個、２個、４個、８個、
このあたりが定番ね。
『Tabキー』１回で済ますのも一般的ね。
普通の場合はTabキー１回で半角スペース８個分になるかしら。
自分で好きな量だけ選んで使いなさいな。





まあインデントについてはこんなところかしら。
これは弾幕風のスクリプトに限らずプログラミング全体の作法だから、
蔑ろにするわけにはいかないわね。
それじゃ今回の話はこれでおしまい。
それじゃまたね〜
