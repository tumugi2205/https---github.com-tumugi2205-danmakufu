閑話休題　コメントアウトについて



あらいらっしゃい。
此処では弾幕作成とは直接的に関係ない話をさせてもらうわ。
まあそんなに気張らなくても大丈夫よ。


さて、今回のお話はコメントアウトについて。
コメントアウトとは、プログラムに直接影響せずに好き勝手書けるようにできるものよ。
厳密な定義とは違うけれど、使い道としてはそんな認識でいいわ。
じゃあコメントアウトの使用例を見てみましょう。
まずは『study003.txt』の自機狙いを絶えず撃つスクリプトを使って説明しましょう。 


〜　前略　〜

  task mainTask {
    loop(60) {yield;}

    loop {
      CreateShot01(GetX, GetY, 2, GetAngleToPlayer, RED01, 0);
      loop(60) {yield;}
    }
  }
}

上の例はまだコメントアウトを使っていない状態ね。
じゃあ適当にコメントアウトを使ってみましょう。 


〜　前略　〜

/*『study003.txt』のメイン処理よ
  基本中の基本の処理ね*/
  task mainTask {
    loop(60) {yield;}	//１秒待つ処理よ

    loop {	//無限ループのはじまりはじまり〜
      CreateShot01(GetX, GetY, 2, GetAngleToPlayer, RED01, 0);	//自機狙いの弾を撃つわ
      loop(60) {yield;}	//ここも１秒待つ処理ね
    }
  }
}




さて、スクリプト中に適当なコメントを書いてみたわ。
普通に全角文字を書くと、エラーが出て動かないわね。
でもこのスクリプトだったらちゃんと動くわ。
コメントアウトをしているからね。
ではコメントアウトのやり方の説明をしましょう。
コメントアウトのやり方には２種類の方法があるわ。






では１つめの方法。
『/*』と『*/』で括る方法よ。
/*と*/の間に書かれている記述は、プログラム処理で全て無視されるわ。
この間になにを書いても問題なしってワケ。
だからプログラムがどんな処理をしているのかメモすることもできるわ。
これが主なコメントアウトの使用目的ね。
で、２つめの方法、『//』を使う方法。
//から、行の最後までの間がコメントアウト対象になるわ。
短いコメントだったら//の方が使い勝手が良いでしょうね。



それでコメントアウトの使用目的だけど、さっき言ったとおりメモがメインよ。
スクリプトがどんな動きをするのかわかりにくい部分は積極的に使った方が良いでしょう。
またスクリプトが大規模になってきたなら区切りの良さそうなところにも
メモを置いた方が後々の助けになるでしょう。
他にも自分が好きなようにコメントアウトを使いなさいな。
でもある程度他人に見てもらうことを意識して書きなさいね。



他の使い方といったら、スクリプト文を一時的に退避させたい場合かしらね。
スクリプトを改変したいけど、もしかしたら元に戻せないかもしれない･･････
そんな不安があるときは、書き換えるところの文をコピーして貼り付けて、
片方をコメントアウトすればいいわ。
比較しながら作業できるし、修復も楽だし、問題なくスクリプトを動作させることができるし。
まあ他にもうまい使い方が思いついたら好きな使い方をするといいわ。


じゃあ、今回はこんなところかしらね。
コメントアウトを使わなくてもスクリプトは使えるけど、使った方が色々と便利よ。
自分で作ったスクリプトでも、ちょっと時間が経っただけで忘れてしまったりするからね。
他人に見せる場合は勿論あった方が良いに決まっているし。
では、また別の場所でお会いしましょう。
